# BackupMonitor API — Kit de Downstream (Exemplo Fictício)

Este repositório **não** contém o código real da aplicação.  
Ele é um **exemplo de governança de downstream** (qualidade, CI/CD, release, observabilidade, maturidade) para o sistema fictício **BackupMonitor API**, usado na disciplina _Análise, Projeto e Desenvolvimento Ágil_.

A ideia é mostrar como um time poderia organizar:

- Fluxo de trabalho com Git e Pull Requests.
- Pirâmide de testes, lint e SAST com *quality gates*.
- Pipeline de CI/CD com estágios mínimos.
- Política de promoção entre ambientes (dev → staging → prod).
- Estratégia de release segura + plano de rollback.
- Observabilidade mínima e métricas DORA.

> Atividade baseada nas orientações do Prof. Edson Vaz Lopes (Unidade 3 — Downstream).
